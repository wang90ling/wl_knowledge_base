# Flutter 面试题库 - 初级 / 中级 / 高级 分级版

> 适合分阶段刷题、面试准备和团队培训。题目按难度分级，尽量贴近真实面试，且每题都给出标准回答方向。

# 初级篇

## 1. Flutter 和原生开发有什么区别？
**标准答案：**
Flutter 使用自己的渲染引擎绘制 UI，跨平台一致性更高；原生则分别使用 iOS/Android 各自的技术栈。Flutter 更强调一套代码多端运行、开发效率高、UI 一致性强。

## 2. Widget 是什么？
**标准答案：**
Widget 是 Flutter 中 UI 的配置描述对象，轻量、不可变。UI 的变化本质上是根据新的 Widget 配置重新构建界面。

## 3. StatelessWidget 和 StatefulWidget 的区别是什么？
**标准答案：**
StatelessWidget 不保存可变状态，适合静态展示；StatefulWidget 保存可变状态，状态改变后可以通过 `setState` 刷新页面。

## 4. setState 的作用是什么？
**标准答案：**
`setState` 会通知 Flutter 当前页面需要重新 build，让界面根据最新状态重新生成。

## 5. BuildContext 是什么？
**标准答案：**
BuildContext 可以理解为 Widget 在树中的位置引用，用来访问上层依赖、路由、主题、媒体信息等。

## 6. Flutter 中如何跳转页面？
**标准答案：**
最常见的是使用 `Navigator.push` 和 `Navigator.pop`。也可以使用命名路由进行统一管理。

## 7. 如何传递页面参数？
**标准答案：**
常用方式是通过构造函数传参，进入页面时把参数直接传进去，清晰明确。

## 8. Future 和 Stream 的区别是什么？
**标准答案：**
Future 表示一次性异步结果；Stream 表示连续事件流，适合实时监听和多次数据变化。

## 9. 为什么 build 方法里不建议发请求？
**标准答案：**
因为 build 可能被多次调用，把请求写在里面容易造成重复请求、重复刷新甚至循环。

## 10. 为什么要判断 mounted？
**标准答案：**
因为异步任务结束时页面可能已经销毁。如果这时再 setState，可能会报错，所以需要先判断 mounted。

## 11. Flutter 中常用的布局组件有哪些？
**标准答案：**
常用的有 `Row`、`Column`、`Container`、`Padding`、`Expanded`、`Stack`、`ListView`、`GridView` 等。

## 12. const 有什么作用？
**标准答案：**
const 可以让编译期优化对象复用，减少重复创建，提高性能。

# 中级篇

## 1. Flutter 的 Widget、Element、RenderObject 分别是什么？
**标准答案：**
- Widget：UI 配置描述
- Element：Widget 的实例和挂载节点
- RenderObject：负责布局、绘制和命中测试

这三层共同完成 Flutter 的 UI 渲染。

## 2. Flutter 为什么性能通常不错？
**标准答案：**
因为 Flutter 自己负责渲染，减少了和原生系统频繁通信的成本，同时采用了高效的组件化和局部更新机制。

## 3. Provider 的核心思想是什么？
**标准答案：**
Provider 是一种依赖注入和状态订阅方案，让上层可以提供状态，子组件按需监听变化。

## 4. 你会怎么选状态管理方案？
**标准答案：**
- 简单页面：setState
- 中小项目：Provider
- 中大型复杂项目：Riverpod 或 Bloc

要结合项目复杂度和团队经验选型。

## 5. Repository、Service、ViewModel 有什么区别？
**标准答案：**
- Service：封装网络、存储、第三方能力
- Repository：做业务数据组合和抽象
- ViewModel：管理页面状态和交互逻辑

## 6. 如何避免页面因为状态变化频繁重建？
**标准答案：**
可以拆分组件、使用 const、精准订阅局部状态、避免大范围监听整个页面。

## 7. 你怎么处理接口请求失败？
**标准答案：**
会根据失败类型分别处理：网络错误、服务端错误、业务错误、鉴权错误。页面上一般要展示加载态、失败态、空态和重试按钮。

## 8. 401 和 403 的区别是什么？
**标准答案：**
401 是未认证，通常是没登录或 token 失效；403 是已认证但没有权限。

## 9. token 一般怎么处理？
**标准答案：**
token 通常存储在本地，请求时统一加到 header 中，格式一般是 `Bearer xxx`。如果 token 失效，需要清理并跳转登录页。

## 10. 如何封装网络层？
**标准答案：**
一般分三层：
- NetworkClient：基础请求和错误处理
- ApiService：具体接口封装
- Repository：业务组合

这样结构清晰，方便维护和测试。

## 11. 为什么要做数据缓存？
**标准答案：**
缓存可以减少重复请求，提升首屏速度，并在弱网和离线场景下提供兜底。

## 12. 如何优化长列表？
**标准答案：**
- 使用 builder 懒加载
- item 结构轻量
- 图片做缓存和压缩
- 分页加载
- 减少 item 内的复杂异步和重建

## 13. 怎么优化首屏加载？
**标准答案：**
首屏只加载关键数据，非关键内容延迟加载，配合缓存和占位图，减少复杂初始化逻辑。

## 14. Flutter 中怎么释放资源？
**标准答案：**
像 `TextEditingController`、`ScrollController`、`AnimationController`、流订阅等都需要在 dispose 中释放或取消订阅。

## 15. 如何做登录拦截？
**标准答案：**
一般在网络层统一识别 401，抛出未授权异常，清理本地登录态后跳转登录页。

# 高级篇

## 1. Flutter 的渲染流程是怎样的？
**标准答案：**
Flutter 从 Widget 构建 Element 树，再映射到 RenderObject 树，随后经历布局、绘制和合成，最终由引擎渲染到屏幕。

## 2. 什么情况下会导致严重卡顿？
**标准答案：**
- build 太重
- layout 太复杂
- paint 过多
- 图片过大
- 动画中频繁重建
- 状态监听范围过大

## 3. 你如何定位卡顿问题？
**标准答案：**
先判断是 build、layout 还是 paint 问题，再结合 DevTools 和性能面板看帧耗时、内存和重绘区域。

## 4. 如何优化复杂列表页面？
**标准答案：**
- 把 item 拆小
- 避免 item 内层过多嵌套
- 使用缓存图片
- 分离局部状态
- 必要时用分页和预加载
- 使用 RepaintBoundary 隔离复杂绘制区域

## 5. 你怎么设计一个大项目的目录结构？
**标准答案：**
会按职责拆分：pages、viewmodels、repositories、services、models、core、widgets 等，让页面、业务、网络、工具互相解耦。

## 6. 为什么页面不要直接调接口？
**标准答案：**
因为页面直接调接口会让 UI、业务和网络耦合太高。通常应该通过 ViewModel 或 Repository 间接调用，方便测试和复用。

## 7. 如何统一处理全局错误？
**标准答案：**
网络层抛出统一异常，业务层根据错误类型做不同处理，UI 层展示统一的错误页面或提示。401 还要单独做登录跳转。

## 8. 如何处理“接口成功但业务失败”？
**标准答案：**
要把 HTTP 成功和业务成功分开看。接口 200 只表示网络通了，真正业务是否成功还要看返回 code、msg 等字段。

## 9. 如何实现离线兜底？
**标准答案：**
可以结合本地缓存和最近一次成功数据，在网络失败时展示缓存内容，同时提示用户当前是离线/弱网状态。

## 10. 你会如何设计登录态管理？
**标准答案：**
登录态应统一管理 token、用户信息、过期策略、退出清理、401 失效跳转等，避免分散在多个页面里。

## 11. 什么是 RepaintBoundary？什么时候用？
**标准答案：**
RepaintBoundary 用于隔离绘制区域，避免父层频繁重绘影响子层。适合复杂动画、局部更新、列表中较复杂的独立区域。

## 12. 为什么 const、final、immutable 很重要？
**标准答案：**
不可变对象更容易预测和优化。Flutter UI 里大量使用不可变对象，能减少状态混乱，也便于框架做复用优化。

## 13. Flutter 中如何做性能监控？
**标准答案：**
可以通过 DevTools、日志埋点、关键接口耗时统计、首屏耗时统计、FPS 监控等方式进行。

## 14. 如何设计一个可测试的 Flutter 架构？
**标准答案：**
把 UI、业务逻辑、数据访问分离，核心逻辑放在可独立测试的层。接口和存储依赖通过注入方式替换，方便 mock。

## 15. 你会怎么处理 401 自动跳登录？
**标准答案：**
在网络层识别 401，抛出统一未授权异常；在上层捕获后清理本地 token 和用户信息，再跳转登录页，避免用户看到原始错误页面。

## 16. 为什么说 Flutter 不是“只会写页面”就够了？
**标准答案：**
因为真正项目里还要处理状态管理、数据流、性能、登录态、缓存、原生交互、测试和工程化。页面只是表层，稳定性和可维护性才是核心。

## 17. Flutter 项目中最容易被忽略的工程问题有哪些？
**标准答案：**
- 网络层没有统一错误处理
- 登录态分散管理
- 页面状态过于臃肿
- 列表和图片性能差
- 缺少空态、失败态、重试态
- 没有统一组件规范

## 18. 大型 Flutter 团队协作时最重要的是什么？
**标准答案：**
统一规范最重要，包括目录结构、状态管理方式、网络层封装、主题、日志、错误处理和组件规范。这样多人协作不会混乱。

## 19. 如何处理页面中的复杂业务流程？
**标准答案：**
把流程拆成状态机或明确的状态流转，避免在页面里堆很多 if/else。必要时把复杂流程下沉到 ViewModel 或业务控制器中。

## 20. 你如何总结一个成熟 Flutter 开发者的能力？
**标准答案：**
成熟的 Flutter 开发者不仅会写 UI，还能做好状态管理、架构设计、性能优化、错误兜底、网络封装、登录态处理、原生协作和测试。

# 补充题：实战追问

## 1. 如果接口超时了你怎么处理？
**标准答案：**
提示网络异常，允许重试，必要时使用缓存数据兜底，并记录请求耗时便于排查。

## 2. 如果用户频繁点击按钮导致重复请求怎么办？
**标准答案：**
可以通过按钮置灰、loading 状态、防抖、节流或请求锁来避免重复提交。

## 3. 如果页面显示空白，你怎么排查？
**标准答案：**
先看数据有没有回来，再看是否 setState，最后看布局是否被约束问题或错误条件分支挡住。

## 4. 如果图片加载很慢怎么办？
**标准答案：**
使用缩略图、缓存、占位图，并控制图片分辨率和列表中同时加载的数量。

## 5. 如果用户退出登录后又返回到了受保护页面怎么办？
**标准答案：**
应当在登录态失效时统一重定向到登录页，退出登录时清空栈，避免回退回受保护页面。

一、问题分析：为什么不能直接用 mipmap 里的图片？
核心原因：Flutter 和 Android 原生是两个独立的资源系统。


Plain Text

┌─────────────────────────────────────────────────────┐
│  Flutter 层 (Dart)                                  │
│  ┌───────────────────────────────────────────────┐  │
│  │  Flutter Assets 资源                           │  │  ← Flutter 只能访问这里
│  │  (pubspec.yaml 声明的 assets/ 目录)             │  │
│  └───────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────┤
│  Android 原生层 (Kotlin/Java)                        │
│  ┌───────────────────────────────────────────────┐  │
│  │  mipmap/drawable 资源                          │  │  ← 只有 Android 原生存活代码能直接访问
│  │  (应用图标、通知图标、启动图等)                    │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
二、三种方案对比
方案	实现方式	优点	缺点	推荐度
方案一：复制到 Flutter assets	图片复制到 assets/images/，pubspec.yaml 声明，用 Image.asset()	跨平台、性能好、维护简单	需要复制一份	⭐⭐⭐⭐⭐
方案二：Platform Channel	写原生代码读 Bitmap 转 bytes 返回 Flutter	能动态获取任意原生资源	实现复杂、性能差	⭐⭐
方案三：继续用 IconData	保持现有字体图标方案	体积小、可着色、矢量	无法自定义	⭐⭐⭐⭐
三、已完成的实现（方案一）
1. 创建资源目录
   assets/images/ - 已从 mipmap-xhdpi 复制 8 个常用图标

2. pubspec.yaml 声明
   pubspec.yaml


YAML

assets:

_ServiceItem - 服务网格图标组件

四、使用方式
用 IconData（原有方式）：


dart

_ActionItem(
icon: Icons.receipt_long_rounded,
label: '订单',
gradient: ...,
onTap: () {},
)
用本地图片（新方式）：


dart

_ActionItem(
iconAssetPath: 'assets/images/ic_me_ser_jdjl.webp',
label: '订单',
gradient: ...,
onTap: () {},
)
五、注意事项
webp 格式支持 - Flutter 原生支持 webp，无需额外配置
图标着色 - Image.asset 的 color 参数可以像 Icon 一样着色，保持视觉统一
图片尺寸 - 建议使用 2x/3x 图（放 assets/images/2.0x/、assets/images/3.0x/）以适配不同分辨率
新增图片 - 只需把图片放到 assets/images/ 目录即可，pubspec.yaml 中声明了目录会自动包含
